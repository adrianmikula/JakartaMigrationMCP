# Common Issues and Gotchas

This document tracks persistent issues encountered during development and their solutions.

## IntelliJ Plugin Compatibility

### Issue: untilBuild Attribute Automatically Added by Gradle Plugin
**Problem**: The IntelliJ Gradle plugin (`org.jetbrains.intellij`) automatically adds a default `untilBuild="233.*"` attribute to the generated `plugin.xml` when the `patchPluginXml` configuration is not explicitly set. This prevents the plugin from loading in IntelliJ versions newer than 233, even when the plugin should be compatible with all future versions.

**Root Cause**: The IntelliJ Gradle plugin has a default value for `untilBuild` of `233.*` when the property is not explicitly set to something other than null/empty. Simply not setting the property in `gradle.properties` does NOT result in no attribute - it results in the default `233.*`.

**Symptoms**:
- Plugin fails to load in IntelliJ versions > 233
- Marketplace validation shows compatibility range errors
- Users see "Plugin is incompatible with this IntelliJ version" messages

**Solution**: 
Add explicit `patchPluginXml` configuration to `build.gradle.kts`:
```kotlin
patchPluginXml {
    sinceBuild.set(providers.gradleProperty("intellij.sinceBuild").orElse("233"))
    untilBuild.set(providers.gradleProperty("intellij.untilBuild").orElse(""))
}
```

This generates:
```xml
<idea-version since-build="233" />
```

Instead of the problematic:
```xml
<idea-version since-build="233" until-build="233.*" />
```

**Verification**: Create tests to verify the generated `plugin.xml` does not contain `until-build` attribute after building the release ZIP using the `publishPlugin` task.

## Windows Path Handling

### Issue: PathMatcher Glob Failures
**Problem**: The Java `PathMatcher` with glob patterns (e.g., `glob:**/*.java`) behaves inconsistently on Windows because of backslash (`\`) separators. Relative paths generated by `path.relativize()` often use the system separator, causing matches to fail if the glob pattern uses forward slashes.

**Solution**: 
- Normalize all paths to use forward slashes (`/`) before matching.
- Use a custom regex-based matching for more robust cross-platform support.
- Example normalization: `path.toString().replace('\\', '/')`.

## Logging and Terminal Output

### Issue: Missing Log Output in Tests
**Problem**: By default, SLF4J may default to a "No-Operation" (NOP) logger if no implementation is found on the classpath. Furthermore, Gradle does not show standard output/error from tests by default.

**Solution**:
- Add `testImplementation("org.slf4j:slf4j-simple:2.0.9")` to `build.gradle.kts`.
- Configure Gradle to show standard streams:
  ```kotlin
  tasks.test {
      useJUnitPlatform()
      testLogging {
          showStandardStreams = true
      }
  }
  ```

### Issue: PowerShell Encoding for Redirected Output
**Problem**: Redirecting command output to a file in PowerShell (e.g., `gradlew test > output.txt`) often uses `UTF-16LE` encoding, which may be unreadable by AI tools or other utilities.

**Solution**: Use `-Encoding UTF8` when reading files or use specific PowerShell redirection commands if possible. When running via AI tools, ensure the reading tool can handle the encoding or force UTF8.

## SQLite Database Persistence

### Issue: Generated Keys Not Supported
**Problem**: The SQLite JDBC driver does not consistently support `Statement.RETURN_GENERATED_KEYS`. Attempting to call `stmt.getGeneratedKeys()` can result in a `SQLFeatureNotSupportedException`.

**Solution**: Immediately after an `INSERT` statement, execute a query to get the last inserted row ID:
```java
try (Statement idStmt = conn.createStatement();
     ResultSet rs = idStmt.executeQuery("SELECT last_insert_rowid()")) {
    if (rs.next()) {
        long id = rs.getLong(1);
        // ...
    }
}
```

### Issue: Nested Connections and Transactions
**Problem**: Performing queries within a loop that uses its own connection management can lead to locking or incomplete results in SQLite, especially if transactions aren't handled carefully.

**Solution**: Reuse the same connection for related operations or ensure results are fully drained into memory before opening secondary connections.
